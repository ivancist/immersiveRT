<objective>
Wire the Plan-01 WebXR module into the live phone sensor pipeline so that, on ARCore-capable Android, camera-assisted 6DOF pose becomes the primary position source, while every unsupported device and iOS keeps the existing ZUPT+Kalman fallback unchanged (D-04). The branch is chosen once at calibration and locked for the session (D-06). Add the D-05 tracking-mode badge to the calibrating screen. Route all camera-derived values through the identical clamp + safeFloat sanitisation used for Kalman output (Pitfall 4). The wire format, desktop decoder, and orientation source are untouched (D-01).

Purpose: Deliver SENS-V2-03 (camera VIO via WebXR) as a drop-in position source without disturbing the proven fallback pipeline or the desktop side.
Output: modified `client/phone.html` (badge) and `client/src/phone.ts` (branch integration).
</objective>
