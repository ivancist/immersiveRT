**Guarded numeric method pattern** (`kalman.ts` lines 57-66):
```typescript
predict(accelMs2: number, dtSec: number): number {
  // T-05-01: guard non-finite inputs before mutating state.
  if (!Number.isFinite(accelMs2)) return this.pos;
  if (!Number.isFinite(dtSec) || dtSec <= 0) return this.pos;

  this.vel += accelMs2 * dtSec;
  this.pos += this.vel * dtSec;
  this.P += this.Q * dtSec;
  return this.pos;
}
```
Apply the same "guard first, return last-known-good value on invalid input" idiom in `webxr.ts`'s per-frame pose read — this is the exact idiom RESEARCH.md's Pitfall 5 (freeze-on-lost) calls for: return last-known-good `{x,y,z}` instead of 0/NaN when `getViewerPose()` returns `null`.

**Sliding-window pure computation pattern** (`zupt.ts` lines 56-92, private `_evict`/`_evaluate`):
```typescript
update(accelMag: number, nowMs: number): boolean {
  if (!Number.isFinite(accelMag)) {
    this._evict(nowMs);
    return this._evaluate();
  }
  this._window.push({ v: accelMag, t: nowMs });
  this._evict(nowMs);
  return this._evaluate();
}

private _evict(nowMs: number): void {
  while (this._window.length > 0 && nowMs - this._window[0].t > this.windowMs) {
    this._window.shift();
  }
}
```
Directly reusable shape for D-03's rolling ~300ms gestureDisplacement ring buffer (RESEARCH.md Pattern 3) — same push/evict/evaluate structure, same `windowMs` field convention, same "public tunable threshold field" style as `zupt.ts`'s `adaptiveThreshold`.

## Shared Patterns

### Guarded numeric input (NaN/Infinity safety)
**Source:** `client/src/sensor/kalman.ts` lines 58-60, `client/src/sensor/encode.ts` `safeFloat()` lines 44-47
```typescript
export function safeFloat(v: number | null | undefined, fallback = 0): number {
  if (v == null || !isFinite(v as number)) return fallback;
  return v as number;
}
```
**Apply to:** Every camera-derived x/y/z value in `webxr.ts` before it reaches `phone.ts`'s packet-construction step, and again via the existing `safeFloat()` call already wrapping every field in `encodePacket()` (`encode.ts` lines 97-123) — this is a double-guard by design (Pitfall 4), not redundant.
