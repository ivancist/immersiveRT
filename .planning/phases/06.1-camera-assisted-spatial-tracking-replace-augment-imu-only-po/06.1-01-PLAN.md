<objective>
Build the self-contained Android WebXR pose-tracking module `client/src/sensor/webxr.ts` plus the ambient WebXR TypeScript declarations it needs and a full unit-test suite for its jsdom-testable pure surface. This module is the drift-resistant replacement for IMU-only dead-reckoning; it exposes support detection, a per-frame tracking-state → driftConfidence mapping (D-02), a rolling ~300ms gestureDisplacement tracker (D-03), and freeze-on-lost last-known-good retention (Pitfall 5 / D-06). It calls the raw `navigator.xr` API directly — no Three.js, no new npm package (phone.ts has no renderer, per RESEARCH.md).

Purpose: Isolate all WebXR logic behind a small tested API so Plan 03 can wire it into the live sensor pipeline with a single-line branch, leaving the existing ZUPT/Kalman fallback untouched.
Output: `client/src/webxr.d.ts`, `client/src/sensor/webxr.ts`, `client/tests/webxr.test.ts`.
</objective>
