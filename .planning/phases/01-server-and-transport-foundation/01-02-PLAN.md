<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → QUIC:4433 | Untrusted WebTransport connections arrive here; accept loop must not panic |
| filesystem → TLS identity | Private key loaded from certs/ at startup; must never be committed |
| WebTransport client → echo handler | Client-supplied JSON must not cause panic on malformed input |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04 | Information Disclosure | certs/localhost+2-key.pem | high | mitigate | Task 1 verifies `.gitignore` has `certs/` before cert generation; `Identity::load_pemfiles` reads from filesystem only at startup — key never serialized or logged |
| T-01-05 | DoS | wt_server accept loop | high | mitigate | Task 2: each accepted connection dispatched to `tokio::spawn`; errors in connection tasks logged with `tracing::error!` and do not re-enter or kill the accept loop; unbounded connection growth is a Phase 2 concern (max-connections limit) |
| T-01-06 | Tampering | echo handler JSON parsing | medium | mitigate | Task 2: use `serde_json::from_slice` with typed `EchoMessage` deserialization — malformed JSON returns `Err`, which is logged and the connection is dropped cleanly (no panic, no unwrap) |
| T-01-07 | Spoofing | TLS handshake (mkcert CA) | low | accept | mkcert CA is dev-only; production will use a trusted CA (Let's Encrypt) in Phase 2 Docker deployment; Chrome dev flag makes the trust boundary explicit |
</threat_model>
