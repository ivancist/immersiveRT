<objective>
Extend main.rs with the TURN credential HTTP endpoint. This satisfies INFRA-04: ephemeral credentials generated at connection-start (not page load). The endpoint is served by axum on a dedicated HTTP port (HTTP_PORT, default 8081) — a plain HTTP server for Phase 2 (TLS on the TURN endpoint is a Phase 4 concern when browser clients need HTTPS).

Purpose: The TURN credential endpoint completes the coturn integration. Clients call GET /turn-credentials right after connecting to the signaling server to obtain short-lived HMAC-SHA1 credentials that coturn will accept. The TURN_SHARED_SECRET is a required env var — the server refuses to start without it.

Output: main.rs extended with AppState, axum Router with /turn-credentials route, HTTP server task in tokio::try_join!, and a unit test proving the handler logic is correct.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → main.rs | TURN_SHARED_SECRET flows from process environment into AppState — never logged, never serialized to client |
| client → /turn-credentials | Any client on the network can request TURN credentials — no authentication in Phase 2 (Phase 3 adds session auth) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-11 | Information Disclosure | TURN_SHARED_SECRET hardcoded in source or logged | critical | mitigate | Required env var via std::env::var("TURN_SHARED_SECRET")?  — no default, no logging; server fails to start if absent; the only place the value appears is in AppState (in-memory only) |
| T-02-12 | Tampering | TURN credential TTL too long — stale credential extends replay window | low | accept | Default TTL 300s (5 min); INFRA-04 requires generation at connection-start (not page load) ensuring freshness regardless of TTL; short TTL is the primary defense per RESEARCH.md Pitfall 1 |
| T-02-13 | Elevation of Privilege | /turn-credentials unauthenticated — any client can obtain TURN relay allocation | medium | accept | Phase 2 scope; TURN relay is data-only (not video/audio) and low-bandwidth (IMU packets); TURN cost risk is low for this use case; Phase 3 adds session auth |
</threat_model>
