## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env → docker-compose | TURN_SHARED_SECRET flows from .env into container environment — must never reach git |
| coturn → internet | coturn binds directly to host interfaces with network_mode: host — exposed to any network the host is connected to |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-07 | Information Disclosure | TURN_SHARED_SECRET committed to git via docker-compose.yml or turnserver.conf | high | mitigate | .env file (gitignored) holds the actual secret; docker-compose.yml uses ${TURN_SHARED_SECRET} substitution; .env.example committed with only a placeholder value; verify .env is in .gitignore |
| T-02-08 | Spoofing | coturn exposed without fingerprint directive — allows TURN packet spoofing | high | mitigate | fingerprint directive in turnserver.conf authenticates all TURN protocol messages |
| T-02-09 | Denial of Service | coturn ports: block alongside network_mode: host — mapping silently ignored, causing "coturn unreachable" debugging confusion | medium | mitigate | coturn service has NO ports: block; bind addresses/ports controlled exclusively by turnserver.conf and --external-ip command arg (RESEARCH.md Pitfall 2) |
| T-02-10 | Tampering | nginx serving client files over HTTP without TLS on port 8080 | low | accept | Phase 2 scope: the phone/desktop client apps do not exist yet; production deployment will route nginx behind a TLS-terminating proxy; flagged for Phase 4 |
</threat_model>
