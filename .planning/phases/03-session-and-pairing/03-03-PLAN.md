  <action>
    Create the directory docker/nginx/ if it does not exist. Create docker/nginx/nginx.conf with a single server block that:

    Listens on port 80 (HTTP) for local non-phone testing and listens on port 443 with ssl for HTTPS (required for camera-app QR auto-open per D-13 and RESEARCH.md Pitfall 5).

    SSL directives: ssl_certificate /certs/localhost+2.pem; and ssl_certificate_key /certs/localhost+2-key.pem; — these paths match the volume mount that will be added in Task 2.

    Document root: root /usr/share/nginx/html; and index index.html;

    Location block: location / { try_files $uri /index.html; } — this single directive handles SPA routing (D-06): static assets like room.js are served directly ($uri match), all other paths (/room/ABCD, /phone) fall back to index.html.

    Do NOT add a separate location block for /api or similar — the nginx server is static-only; the Rust server handles all WebSocket and WebTransport traffic on ports 9090 and 4433.

    No gzip, no access_log customization, no rate limiting — keep the config minimal for Phase 3. Those are Phase 7+ concerns.
  </action>

  </verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| phone browser→nginx TLS | Phone connects to nginx HTTPS; cert must be trusted on device (mkcert CA) |
| pairing_url HTTPS vs HTTP | If BASE_URL defaults to HTTP, iOS camera app will not auto-open — the QR scan silently fails |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Elevation of Privilege | nginx HTTPS for QR pairing URL | high | mitigate | nginx serves HTTPS on port 8443 with mkcert certs. BASE_URL is a required env var (no HTTP default). SERVER refuses to start without BASE_URL, preventing accidental HTTP QR codes. |
| T-03-08 | Information Disclosure | nginx serving certs directory | medium | mitigate | certs/ volume mounted as :ro (read-only). nginx serves only /usr/share/nginx/html (client/dist). The certs directory is not under the document root — nginx cannot serve cert files as static assets. |
</threat_model>
