<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→join-room handler | Untrusted payload from any WS/WT client claiming a username, room_code, game_type |
| QR scan→pair handler | Untrusted token string arriving from any phone via WS/WT |
| Reconnect→reconnect handler | Untrusted reconnect_token string claiming a slot identity |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Spoofing | pairing_token.rs validate_and_consume | high | mitigate | Single-use tracking via DashMap entry (atomic check-and-insert). TTL = pairing_ttl_secs (default 90s). Constant-time HMAC verification via verify_slice. |
| T-03-02 | DoS | room_registry handle_join | medium | mitigate | Max 8 slots enforced server-side (D-08). Room code check before slot scan prevents slot enumeration. One join per connection (WS/WT connection semaphore already limits total connections). |
| T-03-03 | Information Disclosure | pairing_token.rs HMAC verify | medium | mitigate | mac.verify_slice(&sig_bytes) is constant-time. String equality (==) is forbidden for HMAC comparison (RESEARCH.md Anti-Patterns). |
| T-03-04 | Tampering | room_registry handle_join username | medium | mitigate | Username: trim, 1–64 chars, printable ASCII only (strip control chars). Reject empty after sanitization. Stored username is only used for broadcast display, not auth. |
| T-03-05 | Spoofing | room_registry handle_reconnect | medium | mitigate | Reconnect token is opaque random 32-byte base64; looked up in server-side DashMap. Only the legitimate client who received the token at join time can reconnect. |
| T-03-06 | Tampering | room_registry payload deserialization | low | mitigate | serde_json::from_value returns None on malformed payload; handler returns join-error and drops message (established T-01-06 pattern). |
| T-03-SC | Tampering | cargo install sha2 + rand | high | mitigate | Package legitimacy gate (Task 1 blocking checkpoint). Verify at crates.io/crates/sha2 and crates.io/crates/rand before Cargo.toml is modified. |
</threat_model>
