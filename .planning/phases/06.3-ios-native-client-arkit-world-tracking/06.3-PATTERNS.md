**driftConfidence mapping â€” port directly from `client/src/sensor/webxr.ts`** (lines 43-61, D-07):
```typescript
function readPoseAndConfidence(...): { pos: {...} | null; driftConfidence: 0 | 0.5 | 1 } {
  const pose = frame.getViewerPose(refSpace);
  if (!pose) {
    return { pos: null, driftConfidence: 0 }; // lost
  }
  const p = pose.transform.position;
  if (pose.emulatedPosition) {
    return { pos: { x: p.x, y: p.y, z: p.z }, driftConfidence: 0.5 }; // limited
  }
  return { pos: { x: p.x, y: p.y, z: p.z }, driftConfidence: 1 }; // normal
}
```
D-07 requires the ARKit equivalent: `ARCamera.TrackingState.normal â†’ 1.0`, `.limited(_) â†’ 0.5`, `.notAvailable â†’ 0.0`.

**Freeze-on-loss pattern â€” port directly from `client/src/sensor/webxr.ts`** (lines 78-110, D-06/D-07):
```typescript
private _lastGood: { x: number; y: number; z: number } = { x: 0, y: 0, z: 0 };
private _driftConfidence: 0 | 0.5 | 1 = 0;

ingest(frame, refSpace) {
  const { pos, driftConfidence } = readPoseAndConfidence(frame, refSpace);
  if (!pos) {
    // freeze â€” never mutate last-known-good on tracking loss.
    this._driftConfidence = 0;
    return;
  }
  this._lastGood = pos;
  this._driftConfidence = driftConfidence;
}
```
`ARPoseSource` must hold `_lastGoodPose` state and only update it on a `.normal`/`.limited` (non-frozen) tracking state, never on `.notAvailable`, mirroring this exactly.

```swift
func safeFloat(_ v: Double, fallback: Double = 0) -> Double {
    guard v.isFinite else { return fallback }
    return v
}
```
