# Phase 06.3 Plan 05: ARKit World-Origin Recentering (D-10/D-11) Summary

Added a pure pose-delta hold-still detector and wired it into `ARPoseSource` to auto-recenter the ARKit world origin once at session start (D-10), plus a manual `TransportManager.recenter()` path (D-11) — both re-anchoring via `setWorldOrigin(relativeTransform:)` with zero visible tracking interruption.

## What Was Built

**Task 1 — `HoldStillDetector` (D-10):** A pure, time-parameterized stillness detector. `ingest(px:py:pz:nowMs:) -> Bool` tracks the previous position sample and an accumulated still-window start time; any frame-to-frame Euclidean displacement above `3mm` resets the window, and `isStill` reports `true` once the window has held for `750ms` continuously. Derives stillness ONLY from ARKit-derived position — no CoreMotion/accelerometer input at all, respecting D-01. Fully deterministic via injected `nowMs` (no `Date()`/real-clock dependency inside the type).

**Task 2 — Wiring (D-10 auto + D-11 manual):**
- `ARPoseSource` now owns a `HoldStillDetector` instance and drives it from every `ARSessionDelegate.session(_:didUpdate:)` callback using the already-converted `ARPose.px/py/pz`. Once the detector reports still, `ARPoseSource` calls `recenter()` exactly once, sets `isCalibrated = true` (idempotent guard against re-firing), and invokes `onCalibrationComplete`. A per-frame `onCalibrationProgress: ((Bool) -> Void)?` closure is exposed for a future "hold still to calibrate…" UI indicator (Plan 06's DynamicToast wiring).
- `ARPoseSource.recenter()` (previously a stub) calls `session.setWorldOrigin(relativeTransform: matrix_identity_float4x4)` — verified via source assertion that no code path anywhere in the file re-runs the session with a tracking-reset configuration, which would cause a visible tracking blip (RESEARCH.md Pattern 2 / Anti-Patterns).
- `TransportManager.recenter()` is a new method that simply delegates to `arPoseSource.recenter()` — the manual D-11 path intended for the Plan 08 overlay-menu Recenter button. Documented as single-player (re-zeros only this phone's ARKit origin), additive to and distinct from the desktop's R-key `resetAllPlayerPositions()` (which offsets all connected players via a broadcast correction).

## Verification

- `HoldStillDetectorTests` (3 tests): sustained-still → true after duration, large jump → reset → false (and re-accumulates), deterministic identical results given identical injected timestamps. All pass.
- `ARPoseSourceRecenterTests` (4 tests, new — added since `ARSession`/`ARFrame` cannot be constructed or driven in the iOS Simulator per RESEARCH.md's Validation Architecture constraint): source-assertion checks that `recenter()` calls `setWorldOrigin(relativeTransform:)`, that the file never re-runs the session with a tracking-reset option, that the `ARSessionDelegate` callback path drives a `HoldStillDetector` and reaches `recenter()`, that `TransportManager.recenter()` delegates to `arPoseSource.recenter()`, and that `ARPoseSource` remains default-constructible (constructing it never touches a live `ARSession` — that only happens inside `start()`, gated by `ARWorldTrackingConfiguration.isSupported`).
- Full `immersiveRTTests` suite: **TEST SUCCEEDED** (all existing suites plus the two new ones green, run on iPhone 16 Simulator).
