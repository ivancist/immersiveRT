# Phase 06.3 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | XCTest (existing — `immersiveRTTests` target) |
| **Config file** | `mobile/ios-app/immersiveRT.xcodeproj/project.pbxproj` (existing scheme) |
| **Quick run command** | `xcodebuild test -project mobile/ios-app/immersiveRT.xcodeproj -scheme immersiveRT -destination 'platform=iOS Simulator,name=iPhone 16' -only-testing:immersiveRTTests/<TestClass>` |
| **Full suite command** | `xcodebuild test -project mobile/ios-app/immersiveRT.xcodeproj -scheme immersiveRT -destination 'platform=iOS Simulator,name=iPhone 16'` |
| **Estimated runtime** | ~60-120 seconds (existing 06.2 suite + new Wave 0 additions) |

**Critical constraint (from RESEARCH.md):** ARKit does **not** run in the iOS Simulator (no camera/motion VIO available). Any test exercising a real `ARSession.run(...)` cannot execute in CI/simulator. All new axis-conversion, `driftConfidence` mapping, and touch-capture logic MUST be structured as pure functions (transform → packet fields) so they stay simulator-testable, mirroring `SensorPacketEncoderTests.swift`'s fixture-driven approach. Actual `ARSession` wiring is exercised only by the manual D-16 on-device checkpoint.

---

## Sampling Rate

- **After every task commit:** Run the specific new/changed test file's `-only-testing:` target
- **After every plan wave:** Full `xcodebuild test` suite green (all existing 06.2 tests + new tests)
- **Before `/gsd-verify-work`:** Full suite must be green AND the D-16 manual on-device checkpoint passed (or explicitly re-scoped per D-18)
- **Max feedback latency:** ~120 seconds (full suite)
