<objective>
Add ARKit world-origin recentering: an automatic reset after a hold-still calibration step (D-10, mirroring the web client's `runCalibration`-before-streaming convention) and a manual phone-side recenter control API (D-11, complementing — not replacing — the desktop R-key `resetAllPlayerPositions`). Both use `setWorldOrigin(relativeTransform:)` to avoid the visible tracking interruption a `.resetTracking` restart would cause (RESEARCH.md Pattern 2).

Purpose: Give the phone a clean, drift-free zero point at session start and on demand.
Output: a pure hold-still detector + tests, `ARPoseSource.recenter()` wired to `setWorldOrigin`, and a manual recenter path through `TransportManager`.
</objective>
