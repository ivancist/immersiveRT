<objective>

hold-still calibration scene (D-08), then streams real 36-byte binary packets — driven by the

the wire) end-to-end, and stand up the calibration UX (D-08) that feeds device-adaptive parameters

`client/phone.html`.

## Phase Goal (user story)

**As a** phone player in an active session, **I want to** have my phone's motion and touch fused
on-device (OS-fused orientation, dead-reckoned position, ZUPT-gated gesture displacement) and
streamed to every desktop as a compact 36-byte binary packet at my device's maximum sample rate,
**so that** each desktop can render my movements with low-latency, drift-honest fidelity.

> This plan delivers the thin end-to-end slice of that story: calibrate → stream real 36-byte
> orientation packets to every desktop. Plan 07 deepens it with position, gesture, and touch.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DeviceOrientationEvent / DeviceMotionEvent → sensor pipeline | Browser sensor values feed the packet builder every tick |
| phone → desktop (unreliable WebRTC data channel) | 36-byte packets cross to the Phase 6 decoder |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering / DoS | NaN alpha/beta/gamma from DeviceOrientationEvent | medium | mitigate | `safeFloat` guards the eulerToQuat args; encode.ts additionally safeFloat-guards every field, so a NaN orientation reading cannot produce a poison float16 for Phase 6 |
| T-05-04 | Tampering | reused `_packetBuf` leaking stale bytes across ticks | low | mitigate | encodePacket overwrites all 36 offsets each call (verified in Plan 03); phone.ts reuses the single module-scope buffer and never reads partial state |
| T-05-14 | DoS | `dc.send` throwing on a closing channel aborts the broadcast loop | low | mitigate | broadcastPacket checks `dc.readyState === 'open'` and wraps each send in try/catch so one bad channel cannot stop packets to the others |
| T-05-SC | Tampering | npm installs | low | accept | No installs in this plan (all deps from Plan 01) |
</threat_model>
