## Phase Goal (MVP user story)

**As a** platform developer evaluating motion precision, **I want to** see each connected phone's live orientation and position drive a distinct 3D object on my desktop with SLERP smoothing and diagnostic overlays, **so that** I can directly assess sensor fidelity and phone-to-screen latency before building a real game.

> Derived from ROADMAP.md §Phase 6 Goal + CONTEXT.md `<specifics>`.

This plan completes the precision-evaluation instrumentation: the diagnostic overlays, keyboard controls, and touch/latency legibility that make the scene an evaluation tool, not just a demo.

<objective>
Add the precision-evaluation instrumentation over the working scene: keyboard controls (P/G/A/H/T/D and TAB-hold), the persistent minimal HUD, the per-player numeric HUD panel, the TAB roster overlay with live channel state, the touch-flash emissive response, and the motion trail. All rendering-side effects live inside the existing rAF loop; all keyboard/DOM wiring lives in room.ts.

Purpose: DESK-05's precision-evaluation intent (CONTEXT `<specifics>`) requires the diagnostic aids in D-06/D-07/D-13/D-14/D-15 so the user can read orientation/position/drift, compare position pipelines (P key), see phone-to-screen latency at a glance (touch flash), and inspect connection health (TAB roster) without leaving the 3D view.
Output: a fully instrumented, keyboard-controllable precision-evaluation scene.
</objective>
