## Orientation source

*(Raised as a clarification before the formal gray-area selection — user corrected an assumption drawn from ROADMAP.md's phrasing.)*

**User's input:** "If using position from ARKit give us orientation for free, let use it."
**Resolution:** Use ARKit's camera-transform pose for both position AND orientation — do not run a parallel CoreMotion orientation pipeline. Flagged a research risk: ARKit's axis convention may not match `DeviceOrientationEvent`'s convention used by the existing desktop decode/SLERP pipeline (Phase 6 already hit an axis-inversion bug once, `quick/260710-w83`).
