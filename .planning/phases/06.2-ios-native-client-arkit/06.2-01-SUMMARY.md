## Decisions Made
- Autocreated `xcodebuild` scheme (no explicit shared `.xcscheme` file) was sufficient for `xcodebuild test -scheme immersiveRT -only-testing:...` — verified empirically rather than assumed; no scheme file was added
- `AnyCodable.encode()` matches on `[Any?]`/`[String: Any?]` case patterns; confirmed via a standalone `swiftc` script that Swift's `[Any] → [Any?]` conditional cast always succeeds at runtime, so this correctly handles the `[Any]`/`[String: Any]` values actually produced by the decode path
- `slot` accessor on `SignalingEnvelope` accepts both `Int` and `Double` from the underlying `AnyCodable` for defensive robustness (mirrors `phone.ts`'s broad `typeof === 'number'` check), even though the server always emits a JSON integer
