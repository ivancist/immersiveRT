<objective>
Establish the phase's test infrastructure and the two foundational types every other plan depends on: the signaling wire envelope and the transport abstraction protocol.

This plan is transport parity groundwork only (D-01) — no ARKit, no sensor fusion. It creates: (1) the XCTest target the project currently lacks (VALIDATION.md Wave 0 — `immersiveRT.xcodeproj` has exactly one `PBXNativeTarget` and no test target), (2) the `SignalingEnvelope` Codable type matching the Rust server's `{type, from, to, payload}` JSON shape byte-for-key, and (3) the `SignalingTransport` protocol that lets WebTransport and WebSocket be swapped behind one interface (D-04's dual-path requirement).

Purpose: Unblock every downstream plan. Nothing that carries a unit test can run until the XCTest target exists; no transport/heartbeat/webrtc code can compile until the envelope + protocol exist.
Output: XCTest target, `SignalingEnvelope.swift`, `AnyCodable.swift`, `SignalingTransport.swift`, and their tests.
</objective>
