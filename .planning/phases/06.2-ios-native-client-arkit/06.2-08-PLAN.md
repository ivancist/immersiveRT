<objective>
Replace the placeholder UI with a real pairing/connection flow wired to `TransportManager`, and implement the iOS Wake Lock equivalent (PHONE-07) with correct lifecycle handling.

The existing `ContentView.swift`/`TokenDetailsView.swift` are placeholders (CONTEXT: UI treatment is Claude's discretion beyond proving the connection works). This plan adds an enum-driven session state and a minimal active-session view showing connection status + open-channel count, and handles `isIdleTimerDisabled` — which, unlike the web Wake Lock API's auto-release, persists until explicitly reset, so it must be tied to `scenePhase` (Pitfall 4). No ARKit (D-01).

Purpose: Make the native client usable end-to-end (scan → connect → stream → status visible) and keep the screen awake during a session without leaking the idle-disable state.
Output: `SessionState.swift`, `ActiveSessionView.swift`, updated `ContentView.swift`/`immersiveRTApp.swift`, and session-state tests.
</objective>
