## Bugs found and fixed this session

### WR-12: phone-state races register as first WT stream
- **Root cause**: `sendPhoneState('wake-lock-active')` called before `register` stream opened. Server enforces register-first and closes connection.
- **Fix**: Added `registered` flag in `phone.js`. `sendPhoneState` returns early if `!registered`. Flag set `true` after `register` msg sent; reset `false` on transport close.

### Wake Lock silently rejected on iOS
- **Root cause**: `requestWakeLock()` called after async WebRTC stack — outside user gesture window. iOS rejects silently.
- **Fix**: Moved `requestWakeLock()` call to immediately inside `DeviceMotionEvent.requestPermission()` granted callback, before any async work.

### TLS cert not covering LAN IP
- **Root cause**: `make dev-certs` only generated cert for `localhost 127.0.0.1 ::1`. Phone hitting `https://192.168.0.107:4433` got TLS error 46.
- **Fix**: Updated Makefile `dev-certs` target to auto-detect LAN IP via `ip route get 1` and include it in mkcert invocation.
- **User action needed**: mkcert root CA must be installed on iPhone (Settings → General → VPN & Device Management → trust the CA).

### Server logs invisible
- **Root cause**: `tracing_subscriber::fmt::init()` defaults to ERROR-only without `RUST_LOG`.
- **Fix**: Added `RUST_LOG=info` to `docker-compose.yml` server environment.
