## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| relay-calls-disconnect-for-phone | Relay exit calling `on_client_disconnect` for phone clients starts a 60s hold timer from QUIC drop instant. Ghost QUIC timeout (alive=true) fires hold timer 9s before phone detects drop → reconnect → `invalid_token`. | blocking | `is_phone_client()` guard in `wt_server.rs:350` + `ws_server.rs:358` relay exit — ALREADY implemented. Do NOT remove. |
| slot-not-held-delay | Heartbeat monitor fires at 65s → slot Disconnected → phone reconnects immediately but slot still Connected → `slot_not_held` for ~65s. | advisory | `handle_reconnect` accepts `Connected` OR `Disconnected` slots — already implemented. |
| broker-reject-duplicate | Old broker rejected duplicate registration → new WT/WS relay got Err → connection closed → reconnect loop gave up. | advisory | `broker.register()` is now infallible force-replace — already implemented. |
