## Phase Goal

**As a** third-party game developer, **I want to** install `immersive-rt` and integrate its typed API, **so that** I can build a phone-motion Three.js game without reading platform source.

> Slice: move the battle-tested, dependency-free data/decode/math layer into the SDK as the real source of truth, with its regression tests, and add the hand-written SLERP the tick will use. Demonstrable capability: the SDK's pure layer is green in its new home and preserves the locked wire semantics and drift-honest naming (SDK-05).

<objective>
Extract `types.ts`, `decode.ts`, `playerStore.ts` verbatim into the SDK (D-06), introduce a self-contained `schema.ts` so decode no longer depends on the client's phone-side `encode.ts`, copy `encode.ts` into the SDK to keep the roundtrip regression test intact, and hand-write the plain-object `slerp.ts` (D-07). Relocate the three affected test files from `client/tests/` into the SDK.

Purpose: SDK-05 (drift-honest naming) is already Complete; this extraction must preserve it byte-for-byte, and preserve the ASVS V5 decode guards, while giving the SDK a self-contained, engine-agnostic data layer with zero `three` dependency.
Output: SDK-owned pure modules + relocated regression tests + new slerp module/test, all green.
</objective>
