## Phase Goal

**As a** third-party game developer, **I want to** receive live phone sensor data and player-lifecycle events through the SDK, **so that** each connected phone drives my game with no server relay and no decode code of my own.

> Slice: the live data path. After this plan real phone packets flow phone→SDK→store→tick→getPlayerInput/imuUpdate, and playerJoin/playerLeave/playerReconnect fire from real RTC lifecycle. This is the security-critical extraction — the guard-first decode chain moves verbatim.

|----------|-------------|
