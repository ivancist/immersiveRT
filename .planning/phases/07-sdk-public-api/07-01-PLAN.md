## Phase Goal

**As a** third-party game developer, **I want to** install the `immersive-rt` npm package and integrate its typed imperative + event API (`getPlayerInput`, `on(...)`), latency overlay, and raw-orientation opt-in, **so that** I can build a Three.js multiplayer game driven by phone motion without reading the platform's source code.

> This is a structural extraction + packaging phase. A "vertical slice" here means each plan lands a tested, demonstrable slice of the public SDK surface rather than a UI→API→DB feature stack. This first plan delivers the thinnest end-to-end proof: a buildable, installable, declaration-emitting package.

<objective>
Convert the repo root into an npm workspace and scaffold the `packages/immersive-rt/` package so it builds in Vite library mode (ESM-only) and emits bundled TypeScript declarations. No SDK logic yet — only the packaging pipeline that every later plan builds on.

Purpose: SDK-01 requires a publishable package with TypeScript types for all public surfaces. The packaging pipeline (npm workspace + Vite `build.lib` + `vite-plugin-dts`) is net-new to this repo and must be proven before extracted logic lands. Directly implements CONTEXT.md D-04 (build tooling is Vite library mode — `build.lib` entry, reusing the project's existing Vite toolchain rather than a new bundler) and D-05 (module format is ESM only, matching `client/`'s `"type": "module"` — no CJS dual-build).
Output: Root workspace manifest, `packages/immersive-rt/` package config, a buildable skeleton `src/index.ts`, and a `.gitignore` that ignores root `node_modules/`.
</objective>
