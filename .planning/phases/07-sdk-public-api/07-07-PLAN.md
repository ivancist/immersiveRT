## Phase Goal

**As a** third-party game developer, **I want to** see the existing desktop game harness driven entirely by the `immersive-rt` package, **so that** I have proof the SDK integrates end-to-end without any transport/decode code in the game itself.

> Final slice: make `client/` a consumer of the SDK (D-06 — "client stops owning this logic and becomes a consumer"). This is the live end-to-end proof: the same game harness, now driven by the published package, verified on a real phone→desktop session.

<objective>
Refactor `client/src/room.ts` to drive its transport, join/RTC, and decode entirely through the SDK's `Platform` (connect/joinRoom/reconnect/leaveRoom + on(...) events + onSignal for session UI), removing the duplicated transport/WebRTC/decode code while keeping the lobby/room/QR/roster UI and reconnect-token storage. Rewire `client/src/scene.ts` to read player state from the SDK (`getPlayerInput`/`imuUpdate`) while preserving the W3C→Three.js axis remap in scene.ts (RESEARCH Pitfall 4). Delete the now-SDK-owned `client/src/playerStore.ts` and `client/src/sensor/decode.ts`. Add the `immersive-rt` dependency (plain semver, Pitfall 3) and demonstrate `attachLatencyOverlay()`. Verify live on-device.

Purpose: Proves SDK-01 (a real game integrates against the package) and SDK-04 (overlay live), and completes D-06's extraction by removing the client's ownership of the extracted logic.
Output: room.ts + scene.ts as SDK consumers, client copies deleted, dependency wired, human-verified live session.
</objective>
