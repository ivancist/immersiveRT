<objective>
Fix the cold-start crash where server-1 exits with "Permission denied (os error 13)" on
/certs/localhost+2-key.pem.

Root cause: mkcert generates the private key with chmod 600 (owner-only read). The Docker
container runs as system user 'immersivert' whose UID differs from the host user UID.
The bind-mount ./certs:/certs:ro preserves host file permissions inside the container.
'immersivert' cannot read the 600-mode key → server crashes at startup.

Fix: Create a Makefile that makes `make up` the canonical developer entry point. The `up`
target runs an idempotent `chmod o+r certs/*.pem` before `docker compose up --build`, so
the service user can always read the key regardless of how certs were generated. A
`dev-certs` target wraps mkcert AND sets the correct permissions in a single step.

The `:ro` bind-mount stays unchanged (certs remain immutable inside the container).
No Dockerfile changes required. No gosu/su-exec complexity introduced.

Purpose: Enable cold-start (`docker compose up`) to succeed without any manual chmod step.
Output: Makefile (new), docker-compose.yml (usage comment updated)
</objective>

<task type="auto">
  <name>Task 1: Create Makefile with dev-certs and up targets</name>
  <files>Makefile</files>
  <action>
    Create Makefile in the repo root. Use tabs (not spaces) for recipe indentation — Make
    requires tabs.

    Define the following variables at the top:
    - SHELL := /bin/bash
    - CERT_DIR := certs
    - CERT_FILE := $(CERT_DIR)/localhost+2.pem
    - KEY_FILE  := $(CERT_DIR)/localhost+2-key.pem

    Declare .PHONY targets: dev-certs up down _ensure-certs

    Target `dev-certs`:
    - Check that mkcert is on PATH; print a friendly install hint and exit 1 if not.
    - mkdir -p $(CERT_DIR)
    - Invoke mkcert: mkcert -key-file $(KEY_FILE) -cert-file $(CERT_FILE) localhost 127.0.0.1 ::1
    - chmod o+r $(KEY_FILE) $(CERT_FILE)
    - Print a reminder that certs/ must be gitignored.

    Target `_ensure-certs` (internal, called by `up`):
    - If $(KEY_FILE) does not exist, print "run: make dev-certs" and exit 1.
    - chmod o+r $(KEY_FILE) $(CERT_FILE) — idempotent; fixes certs already generated
      without correct permissions.

    Target `up`: depends on _ensure-certs, then runs `docker compose up --build`.

    Target `down`: runs `docker compose down`.

    Add a help block at the top using a `help` phony target that greps `##` comments from
    the Makefile so `make help` prints available targets. This is the standard self-doc
    pattern: `grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) | sort | awk ...`.

    Annotate dev-certs, up, and down with `## description` comments so help output is useful.

    The chmod step in _ensure-certs runs silently (@chmod) so routine `make up` output
    is not cluttered. The chmod in dev-certs can print nothing or a short note.

    Security note (document in Makefile comment above the chmod line): chmod o+r on a
    private key makes it world-readable on the host. This is acceptable for dev-only
    mkcert certs which are locally trusted and valid only for localhost. Production cert
    handling must use proper secret management — do not apply this pattern to CA-signed
    keys.
  </action>
