# Quick Task 260710-wb4: Prevent Screen Rotation on Phone

## Goal

phone.ts already calls `screen.orientation.lock('portrait')` at DOMContentLoaded but:
- iOS Safari ignores it silently (API unsupported)
- Chrome Android may reject without user gesture / fullscreen

Fix: CSS overlay (works everywhere) + also call lock inside button tap (proper user gesture).

## Tasks

### Task 1: phone.html — add landscape overlay CSS + div

**CSS** (add before `</style>`):
```css
#landscape-overlay {
  display: none;
  position: fixed;
  inset: 0;
  z-index: 9999;
  background: var(--color-bg);
  flex-direction: column;
  align-items: center;
  justify-content: center;
  text-align: center;
  padding: 2rem;
}
@media (orientation: landscape) {
  #landscape-overlay { display: flex; }
}
```

**HTML** (add just before `</body>`):
```html
<div id="landscape-overlay" aria-live="polite">
  <p class="size-heading">Rotate your phone</p>
  <p class="size-body text-secondary mt-md">This controller works in portrait only.</p>
</div>
```
